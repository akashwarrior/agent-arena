# Agent-Arena

# Quick Setup Guide for agent-arena

This guide provides a streamlined setup process for the `agent-arena` monorepo to get you running quickly.

## Prerequisites

Before starting, ensure you have:
- **Node.js** (LTS version)
- **pnpm** package manager
- **PostgreSQL** database

## Installation

1. Clone the repository and install dependencies:
```bash
pnpm install
``` agent-arena:7-11 

## Environment Configuration

Copy the example environment files and configure them:

### Web Application
```bash
cp apps/web/.env.example apps/web/.env
```
Required variables in `apps/web/.env`:
- `BETTER_AUTH_URL` (e.g., `http://localhost:3000`)
- `BETTER_AUTH_SECRET` (random secure string)
- `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` (for OAuth)
- `DATABASE_URL` (PostgreSQL connection string)

### Database Package
```bash
cp packages/database/.env.example packages/database/.env
```
Required variable in `packages/database/.env`:
- `DATABASE_URL` (same PostgreSQL connection string) agent-arena:1-12 agent-arena:1-8 

## Database Setup

Initialize the database schema using Prisma:

1. Generate the Prisma client:
```bash
pnpm db:generate
```

2. Apply database migrations:
```bash
pnpm db:migrate
```

These commands are orchestrated by Turborepo and ensure the database schema is synchronized across the monorepo agent-arena:22-30 .

## Development

Start the development server:
```bash
pnpm dev
```

This command triggers the Turborepo task graph, which:
1. Generates the database client
2. Starts the Next.js development server
3. Runs any other dependent services agent-arena:17-21 

## Architecture Overview

The monorepo consists of:
- **apps/web**: Next.js frontend with authentication
- **apps/agents**: AI agent runtime environment
- **packages/database**: Shared Prisma database layer

All components share the same database schema through the `@repo/db` package agent-arena:1-4 .

## Notes

- The `DATABASE_URL` is defined as a global environment variable in Turborepo, affecting all tasks agent-arena:4-4 
- Database migrations are locked to PostgreSQL to prevent provider mismatches agent-arena:1-3 
- The web app uses shadcn/ui components which can be added via `npx shadcn@latest add [component]` agent-arena:7-13

## Q4
❯ pnpm db:generate
 ERR_PNPM_RECURSIVE_EXEC_FIRST_FAIL  Command "db:generate" not found i am getting this error also i am in the repo root folder
### Answer
The error occurs because `pnpm db:generate` is looking for a script in the root `package.json` that doesn't exist. The `db:generate` script is only defined in the `packages/database/package.json` file agent-arena:5-9 , while the root package.json only contains high-level turbo commands agent-arena:4-8 .

## Correct Commands

Use one of these approaches instead:

### Option 1: Run through Turborepo
```bash
pnpm turbo run db:generate
```

### Option 2: Use the development workflow (recommended)
```bash
pnpm dev
```
This automatically runs `db:generate` first due to the dependency configuration agent-arena:17-21 .

### Option 3: Navigate to the database package
```bash
cd packages/database
pnpm db:generate
```

## Why This Happens

The monorepo uses Turborepo to orchestrate tasks across workspace packages. The `db:generate` task is defined in `turbo.json` agent-arena:22-24  but the actual script implementation exists only in the database package. When you run `pnpm db:generate` from the root, pnpm only checks the root package.json for scripts, not individual workspace packages.

## Available Database Commands

All database commands are available through Turborepo:
- `pnpm turbo run db:generate` - Generates Prisma client
- `pnpm turbo run db:migrate` - Applies database migrations  
- `pnpm turbo run db:deploy` - Deploys migrations in production

These commands are defined in the database package agent-arena:5-9  and orchestrated by Turborepo agent-arena:22-30 .

The `DATABASE_URL` environment variable is globally available to all tasks through Turborepo configuration agent-arena:4-4 , ensuring consistent database connections across the monorepo.

# Agent Arena

A modern full-stack monorepo for AI agent competitions, built with Next.js, Turborepo, and PostgreSQL.

## Quick Start

```bash
# Clone and install dependencies
git clone <repository-url>
cd agent-arena
pnpm install

# Set up environment variables
cp apps/web/.env.example apps/web/.env
cp packages/database/.env.example packages/database/.env

# Start development (automatically sets up database)
pnpm dev
```

## Prerequisites

- **Node.js** (LTS version)
- **pnpm** package manager 
- **PostgreSQL** database

## Project Structure

```
agent-arena/
├── apps/
│   ├── web/           # Next.js frontend application
│   └── agents/        # AI agent runtime environment
├── packages/
│   ├── database/      # Shared Prisma database layer

│   ├── eslint-config/ # ESLint configurations

## Architecture Overview

The monorepo uses **Turborepo** for task orchestration and **pnpm workspaces** for dependency management agent-arena:1-4 . All applications share a common database schema through the `@repo/db` package agent-arena:15-15 .

### Database Layer

- **ORM**: Prisma with PostgreSQL adapter
- **Authentication**: Better Auth integration with OAuth providers
- **Schema**: User management with web3 wallet support
- **Provider**: Locked to PostgreSQL for consistency agent-arena:1-3 

### Frontend

- **Framework**: Next.js with TypeScript
- **UI Components**: shadcn/ui with Radix primitives
- **Styling**: Tailwind CSS
- **Authentication**: Better Auth with Google OAuth

## Development Workflow

When you run `pnpm dev`, Turborepo orchestrates the following tasks:

1. **Database Generation**: Runs `db:generate` to create the Prisma client agent-arena:17-21 
2. **Development Server**: Starts Next.js with hot reloading
3. **Dependency Management**: Ensures all packages are synchronized

The `DATABASE_URL` is globally available to all tasks through Turborepo configuration agent-arena:4-4 .

## Database Schema

The database uses Prisma with the following core models:

- **User**: Authentication profiles with wallet addresses
- **Session**: User login sessions
- **Account**: OAuth provider accounts
- **Verification**: Email verification tokens

All tables use PostgreSQL with proper indexing and foreign key constraints agent-arena:31-79 .

## Troubleshooting

### Common Issues

1. **"Command not found" errors**: Use `pnpm turbo run [command]` instead of `pnpm [command]`
2. **Database connection errors**: Ensure `DATABASE_URL` is set in both `.env` files
3. **Authentication errors**: Verify `BETTER_AUTH_URL` matches your development URL
